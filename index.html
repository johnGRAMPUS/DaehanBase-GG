대한베이스 GG 협업 가이드라인

대한베이스 GG는 한국의 역량 있는 게임 스튜디오가 Base 생태계의 핵심 일원으로 자리 잡고, 글로벌 빌더들과 함께 호흡하며 성장할 수 있도록 지원하는 Web3 게임 빌더 커뮤니티입니다. 함께 학습하고, 실전 빌딩을 공유하며, 한국 게임의 저력을 글로벌 Base 커뮤니티에 각인시키는 것을 목표로 합니다. 대한베이스GG는 그램퍼스가 운영합니다.

⚠️ 꼭 확인해 주세요.
  • 본 프로세스는 ‘대한베이스 GG’와의 파트너십을 위한 전용 협업 가이드이며, Base 팀이나 Base 생태계 전체를 공식 대변하지 않습니다.
  • Base 생태계 온보딩은 누구나 자유롭게 수행할 수 있습니다. ‘대한베이스 GG’와의 협업이 생태계 내에서의 성공을 담보하는 것은 아니며, 협업 여부에 따른 어떠한 불이익도 발생하지 않음을 명확히 밝힙니다.
  • ‘대한베이스 GG’를 통하지 않고 Base 생태계에 자율적으로 합류(셀프 온보딩)하기를 희망하신다면, 본 가이드의 [STEP 2] 단계부터 참조하여 진행하시기 바랍니다.

🎯 가치 제안

  • Global Visibility: Base 생태계라는 거대한 네트워크에서 한국 게임 스튜디오가 고립되지 않도록 돕습니다. 길드 활동을 통해 글로벌 빌더 및 커뮤니티에 프로젝트를 자연스럽게 노출하고 존재감을 각인 시킵니다.
  • Direct Ecosystem Connection: 언어와 시차의 장벽을 넘어, Base 팀, DevRel 및 글로벌 빌더들과 소통할 수 있는 통로가 되어드립니다. 베이스 생태계의 최신 기술 업데이트와 지원 정보를 가장 정확하게 파악하고 대응할 수 있습니다.
  • Practical Knowledge Sharing: 실제 Base 환경에서 작동하는 온체인 게임 경제 설계와 미니앱 최적화 기술을 공유합니다. 시행착오를 줄이고, 글로벌 유저의 취향에 맞는 제품 완성도에만 집중할 수 있습니다.
  • Collective Impact: 길드 내 파트너 빌더와 커뮤니티가 프로젝트의 '글로벌 초기 유저'이자 서포터가 됩니다. 프로젝트 런칭 시 길드의 소셜 네트워크와 채널을 총동원하여 초기 확산을 함께 만들어갑니다.

STEP 1. Screening

가장 먼저 해야 할 일은 '협업의 적기'를 찾는 과정입니다. 스튜디오의 비전이 Base 생태계와 어떻게 조화를 이룰 수 있는지, 그리고 대한베이스GG가 가진 네트워크가 프로젝트에 가장 효과적인 가속 페달이 될 수 있는지를 상호 확인합니다.

1️⃣ 온보딩 폼 신청

대한베이스GG와 협업하여 온보딩을 희망하는 스튜디오, 프로젝트의 기초 정보를 공유합니다. 프로젝트가 가진 매력과 대한베이스GG와 함께 해결하고 싶은 '핵심 과제'를 들려주세요.

  • Project Identity: 프로덕트가 추구하는 재미의 본질과 현재의 빌딩 상태
  • Onchain Exploration: Base 생태계 안에서 새롭게 시도하고 싶은 실험이나 목표
  • Collaboration Goal: 대한 베이스 GG 길드와 함께하며 만들어내고 싶은 구체적인 변화
👉 신청서 링크 바로가기

2️⃣ 킥 오프 미팅

대한베이스GG 운영사인 그램퍼스와의 오프라인 킥오프 미팅을 통해 구체적인 협업 로드맵을 그리고, 지원 체계를 가동합니다. 스튜디오는 이 단계에서 대한 베이스 GG 길드 온보딩을 최종 확정하게 됩니다.

  • 맞춤형 빌딩 가이드: 프로젝트 현재 상태에 최적화된 단계별 실행 가이드를 제공합니다.
  • 빌더 커뮤니티 연결: 글로벌 트렌드를 공유하는 동료 빌더 그룹과의 네트워킹을 시작합니다.

STEP 2. Onchain Identity

Base 생태계는 '빌딩의 결과' 만큼이나 '빌딩의 과정'과 '빌더의 문화'를 중시합니다. 프로덕트 기반인 [프로젝트 계정]과 파운더의 영향력을 확장하는 [파운더 계정]을 이원화하여 빌더 아이덴티티를 구축해 나가는 것을 추천합니다.

1️⃣ 프로젝트 계정

  • Base App, Farcaster, X: 프로젝트 계정을 셋팅하고 주요 Base 프로젝트를 팔로잉 하세요.
  • Builder Code 발급 및 적용: Base.dev를 통해 앱별 빌더 코드를 발급받고 적용하세요. 빌더코드는 앱에서 발생시킨 트렌젝션을 빌더 단위로 귀속시키고 기여도를 측정하게 됩니다.(가이드 확인)

2️⃣ 파운더 계정

  • Base Name: Base App 설치 후 본인의 아이덴티티를 나타내는 Base Name을 클레임합니다. 이는 온체인 프로필의 시작점입니다. (예시 확인)
  • Onchain Reputation: 글로벌 빌더들 사이에서 신뢰를 정량화하기 위해 아래 두 스코어를 관리합니다. 절대적으로 필요한 지표는 아니지만 관리를 권장합니다.
    • Builder Score: 파운더의 지갑 활동, 깃허브 기여도, 보유 자산 등을 종합하여 산출되는 '온체인 이력서' 점수입니다.(점수 확인)
    • Neynar Score: Farcaster 기반의 소셜 평판 점수입니다. 단순 활동량이 아닌 양질의 상호작용을 측정하며, 스팸 계정이 아닌 '진성 빌더'임을 증명하는 척도로 활용됩니다.(Neynar 평점이란)
  • Join the Base Guild: 퀘스트를 클레임하여 베이스 공식 디스코드 내의 권한 및 빌더 역할을 확보합니다. (Guild.xyz/base)
  • 글로벌 빌더 채널: 디스코드 가입 후 각 카테고리별 홍보 채널을 전략적으로 활용하여 글로벌 빌더들과 싱크를 맞춥니다.(공식 디스코드)
  • Builder Network 등록: [Join Builder Network]에 프로젝트를 등록하여 공식적인 빌더 리스트에 이름을 올리고 가시성을 확보합니다.

STEP 3. Product Strategy

Base 생태계에서 성공하기 위해서는 단순히 기존 프로젝트를 온보딩하는 것에 그치지 않고, 베이스 생태계와 커뮤니티의 트렌드를 감지하고 '온체인 고유의 경험'을 프로덕트 전략에 녹이는 것이 중요합니다. 이를 위해 대한베이스GG와의 정기적·실시간 싱크업을 통해 베이스 프로젝트의 내러티브를 정교화하고 최적화된 기술 스택을 매칭하는 것이 좋습니다.

1️⃣ 프로젝트 포지셔닝 및 내러티브 구체화

  • One-Liner & Value Prop: 베이스 생태계는 워낙 넓고 많은 프로젝트들이 있어서 핵심 가치를 한 문장으로 정의하는 것이 중요합니다. 글로벌 유저와 빌더들이 단 3초 만에 프로젝트의 정체성을 이해할 수 있는 강력한 후크를 마련해야 합니다.
  • 주요 베이스 프로젝트의 One-Liner 예시: Tso의 베이스 프로젝트 소개 링크를 참고하여 실제 성공 사례들의 One-Liner를 벤치마크 하세요.

2️⃣ 인기 앱 트렌드

  • Base Leaderboard: [Base Leaderboard]를 통해 한 주간 베이스 생태계에서 주목받았던 앱들을 확인할 수 있습니다. 리더보드에 오르기 위해서는 '빌더 코드' 적용을 꼭 확인해 주세요.

STEP 4. Launch Preparation

베이스 에코시스템 내 가시성을 확보하기 위해 그동안 쌓아온 퍼블릭 빌딩(Public Building) 기록과 데모 영상을 적극적으로 활용하십시오. 베이스 생태계는 빌더와 유저가 함께 섞여있기 때문에 프로덕트의 기능, 엣지를 최대한 부각하는 것이 성공적인 런칭의 핵심입니다.

1️⃣ Base Editorial Style Guide

  • 마케팅 콘텐츠에서 Base Editorial Style Guide를 사용하면 Base의 스토리를 일관성 있게 전달하는데 도움이 됩니다. 또한 Base 코어 팀과 Base 생태계 다른 구성원들로부터 많은 지지를 얻을 가능성이 높아집니다.(Base Editorial Style Guide)

2️⃣ 런칭 보도자료 배포

  • 대한베이스GG와 협업한 프로덕트가 런칭 준비를 마쳤다면 '대한베이스 게임즈 길드와의 협업을 통한 프로덕트 런칭'을 키워드로 한 보도자료 배포가 가능합니다. 세부 사항은 그룹챗에서 팔로업합니다.

3️⃣ 대한베이스 공식 채널 앰플리파이
